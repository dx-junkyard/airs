# LINE聞き取りAIエージェント

## 概要

野生動物の目撃・獣害通報において、住民からの情報収集を自律的に行うAIエージェント。LINEを通じて、自治体職員の代わりにAIが構造化されたインタビューを実施し、対応に必要な情報を漏れなく収集する。

Vertex AI（Gemini）による画像解析・適応型質問生成・状況文の自動生成を組み合わせ、6ステップの対話フローで通報を完結させる。

## 課題

自治体職員へのヒアリングから、従来の電話・窓口対応における3つの課題が明らかになった。

### 1. 必要な情報の聞き取り不足

電話や窓口での聞き取りでは、対応者の経験や状況に左右され、後の対応判断に必要な情報（動物の行動パターン、頭数、移動方向など）が収集しきれないケースが多い。記録のフォーマットも統一されておらず、情報の粒度にばらつきが生じる。

### 2. 鮮明な写真が得られない

通報者から写真を受け取っても、被写体が不鮮明・動物が写っていない・風景のみなど、現場確認に使えない画像が多い。電話越しでは写真の品質を確認する手段がなく、再送依頼もしづらい。

### 3. 現場の状況が具体的に聞けない

「動物を見た」という事実は伝わっても、「何をしていたか」「どの方向に移動したか」「人や建物にどの程度近づいたか」といった具体的な行動の詳細は、聞き手が適切な質問をしなければ引き出せない。結果として、危険度の判断や対策の優先順位付けに必要な情報が不足する。

## ソリューション

### 6ステップの構造化インタビュー

AIエージェントが以下のフローで対話を進め、必要情報を漏れなく収集する。

| ステップ | 内容 | 収集する情報 |
|----------|------|-------------|
| 1. 動物種別 | クイックリプライで選択 | サル/シカ/イノシシ/クマ/その他 |
| 2. 写真収集 | 画像アップロード + AI品質チェック | 現場写真（複数枚対応） |
| 3. 行動詳細 | カテゴリ選択 + 適応型質問 | 具体的な行動・状況の詳細 |
| 4. 日時 | DateTimePickerまたは「今」ボタン | 目撃日時 |
| 5. 場所 | GPS送信または住所入力 | 緯度経度・住所・周辺ランドマーク |
| 6. 確認・送信 | 通報内容のプレビューと修正 | 最終確認 |

### AI画像スクリーニング（課題2の解決）

Vertex AI（Gemini）による3段階の画像品質管理で、使える写真を確実に収集する。

- **1〜2枚目**: 厳格スクリーニング。画質・被写体の判別可否・動物/痕跡の有無を判定
- **3枚目以降**: 緩和スクリーニング。通報者の負担を考慮し、記述生成のみ実行
- **リジェクト時**: 具体的な理由（「被写体が判別できない」「動物が写っていない」等）を提示し再送を促す

合格した写真についてはAIが3つの状況説明文を生成し、通報者が最も近い説明を選択する。

### 適応型質問生成（課題1・3の解決）

行動カテゴリに応じて、AIが最大3問の質問を動的に生成する。

**8つの行動カテゴリと優先観察項目:**

| カテゴリ | 優先的に聞く情報 |
|----------|-----------------|
| 移動 | 方向、速度、移動先 |
| 滞留 | 滞在時間、周囲の様子、人への反応 |
| 接近 | 対象（人/建物/車両）、距離、距離の変化 |
| 採食 | 食べ物の種類、場所、時間 |
| 威嚇 | 行動の種類、対象、人の対応 |
| 逃走 | 方向、きっかけ、速度 |
| 被害 | 被害の種類、程度、場所 |
| その他 | 観察方法、行動の特徴、周囲の環境 |

**質問生成のルール:**

- 既に回答済みの情報は質問しない（重複排除）
- 1問につき2〜6択の選択肢 +「わからない」を必ず含む
- 優先情報が揃った時点でAI自身が質問を打ち切る（`shouldContinue=false`）

### 状況文の自動生成（課題3の解決）

収集したQ&Aの回答から、AIが1〜2文の自然な状況記述を自動生成する。

**生成ルール:**

- 観察ベースの表現のみ使用（「〜のように見えた」「〜の様子だった」）
- 断定的表現は禁止（「確実に〜した」「危険な状態だった」）
- 評価的語句は除外（「危険」「襲った」「恐怖」）
- 動物種別・日時・場所は含めない（別フィールドで管理）

**生成例:**

- 「北東方向へゆっくり歩いて移動しているように見えた」
- 「畑の近くで立ち止まり、何かを食べている様子だった」

通報者は生成された文を確認し、修正がある場合はテキスト入力で訂正できる。

## 効果

### 情報収集の標準化

構造化インタビューにより、対応者の経験に依存せず、一定品質の情報が収集される。動物種別・写真・行動詳細・日時・場所の5要素が必ず揃った状態で通報が完了する。

### 写真品質の向上

AI画像スクリーニングにより、不鮮明・無関係な写真を通報段階で排除できる。リジェクト理由の提示により、通報者自身が「どんな写真が必要か」を理解して再撮影できる。

### 行動詳細の精度向上

適応型質問により、行動カテゴリに応じた観察ポイントを網羅的に収集できる。「動物を見た」だけの通報が、「畑の近くで北東方向に移動中、人を見て逃走した」レベルの詳細な記録に変わる。

### 職員の業務負荷軽減

24時間対応可能なAIエージェントが聞き取りを代行することで、職員は収集済みの情報をもとに対応判断に集中できる。電話対応の時間削減と、聞き直しの手戻りが減少する。
